◆ destroy()

process_manipulation_status destroy ( process_id  id_of_process_to_destoroy)

Function defined as a part of our API for destroying a process.

Destroying a process by accomplishing following tasks:

  • Getting the process context vlock of the process which should be destroyed with the given ID from the processes list
  • Taking a decision whether the process can be destroyed or not by checking these 2 cases:
    • Checking whether a process is already destroyed -> Returning a manipulation status indicating that the process is already destroyed.
    • Checking whether a process is currently running -> Returning a manipulation status indicating the the process is currently running and can not get destroyed consequently.
    • Destroying the process if non of above mentioned cases are applicable
Parameters
[in]process_idid_of_process_to_destoroy (parameter containing the ID for identifying the process to destroy)
[out]process_manipulation_status(status of the manipulation [in this case destruction]-action)
